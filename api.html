Overview
What is NFX OS?

NFX OS is a minimal app runtime running on an ESP32-C3 with a 1.54" SSD1681 e-ink display. A host app manages the launcher and BLE connection. Developer apps plug in as App subclasses.

Sandboxing

Apps operate exclusively through NFXAPI*. Direct GPIO, CPU, memory, or storage access is intentionally withheld to keep apps safe and portable.

Display model

The display is 1-bit (black/white). All drawing happens to an in-memory frame buffer. Call refresh() to push to hardware. Refreshes are slow — batch your draws.

Color convention

c = 0 → white (paper). c = 1 → black (ink). All drawing calls default to black. Pass 0 to draw white (erase).

Coordinate system Origin (0, 0) is the top-left corner. X increases rightward, Y increases downward. The canvas is 200×200 pixels.

App Skeleton

Every NFX app is a single .h file that inherits from App and implements four methods. Create the file in the same Arduino sketch folder as NFX.ino, then add it to the myApps[] array.

App_MyApp.h — minimal skeleton C++ 
#pragma once
#include "NFX_API.h"

class AppMyApp : public App {
private:
    NFXAPI* api;

public:
    String getName() override {
        return "My App";
    }

    void onStart(NFXAPI* a) override {
        api = a;
        api->clear();
        api->text(10, 10, "Hello NFX", 2);
        api->refresh();
    }

    void onLoop() override {
        // called every ~10ms by the OS
        if (api->btnSel()) {
            api->clear();
            api->text(10, 90, "Selected!", 1);
            api->refresh();
        }
    }

    void onStop() override {
        // clean up any state here
    }
};
Registering the app Open NFX.ino and add your app to the launcher array:
#include "App_MyApp.h"
AppMyApp myApp;
App* myApps[] = { &clockApp, &myApp };

App Lifecycle
getName()

Returns the label shown in the OS launcher. Keep it short (≤12 chars) to fit in the menu without clipping.

onStart(api)

Called once when the user launches your app. Store the api pointer, draw your initial frame and call refresh().

onLoop()

Called every ≈10 ms by the OS main loop. This is where you handle input, update state, and decide when to refresh the display.

onStop()

Called when the user holds UP+DOWN to return to the launcher. Release any resources and reset your internal state so the app can be restarted cleanly.

Refresh cost E-ink full refresh is ~2 seconds. Only call refresh() when content has actually changed. Using a dirty-flag variable is the recommended pattern.

Display Control
3 functions
void dispOn()
display
▶
Powers up the e-ink display hardware and initialises SPI. Called automatically by the OS at boot — apps generally don't need to call this unless they have powered the display off intentionally.
void dispOff()
display
▶
Sends the display into deep sleep (hibernate). The last frame remains visible on the e-ink panel. Use when the device will be idle for an extended period to save power.
void refresh()
display
▶
Flushes the in-memory frame buffer to the physical display. This triggers a full e-ink refresh cycle (~2 s). All drawing calls before refresh() are invisible to the user.
Example — dirty flag pattern
bool dirty = false;

void onLoop() {
    if (api->btnUp()) {
        api->clear();
        api->text(10, 10, "UP pressed", 1);
        dirty = true;
    }
    if (dirty) { api->refresh(); dirty = false; }
}

void text(int x, int y, const char* t, uint8_t sz, uint8_t c = 1)
text
▶
Renders a C-string into the frame buffer using the currently active Adafruit GFX font. When no custom font is set, uses the built-in 5×7 pixel font. sz is a pixel-font scale multiplier (1 = normal, 2 = double, etc.).
ParamTypeDefaultDescription
x, yint—Top-left origin for the text baseline.
tconst char*—Null-terminated string to draw.
szuint8_t—Scale factor (1–4 recommended).
cuint8_t1Text color. 0 = white on black.
Example — rendering dynamic values
char buf[16];
snprintf(buf, sizeof(buf), "T: %lu ms", api->now());
api->text(5, 5, buf, 1);
void setFont(const GFXfont* font)  |  void clearFont()
text
▶
Switches to a custom Adafruit GFX proportional font. Include the font header from <Fonts/FreeMono9pt7b.h> (or any other GFX font) and pass its address. Call clearFont() to revert to the built-in bitmap font.
Example
#include <Fonts/FreeSans9pt7b.h>

api->setFont(&FreeSans9pt7b);
api->text(10, 30, "Proportional!", 1);
api->clearFont();
api->text(10, 60, "Back to pixel font", 1);
Note When using proportional fonts the y coordinate is the text baseline, not the top edge. Use textBounds() to measure height before positioning.
void textBounds(const char* t, int x, int y, int16_t* bx, int16_t* by, uint16_t* bw, uint16_t* bh)
text
▶
Measures the bounding box of a string without drawing it. Useful for centering text or drawing a background highlight behind it.
Example — horizontally centered text
int16_t bx, by; uint16_t bw, bh;
api->textBounds("Hello", 0, 0, &bx, &by, &bw, &bh);
int cx = (200 - bw) / 2;

Input
9 functions
Three event types per button Each physical button exposes three event functions: press (rising edge, fires once), held (raw level, true while held), and release (falling edge, fires once). The OS calls sysApi.update() before your onLoop(), so state is always fresh.
bool btnUp()  ·  bool btnUpHeld()  ·  bool btnUpRel()
input
▶
btnUp() — Returns true on the single loop tick when UP is first pressed (rising edge). Use for navigation and discrete actions.

btnUpHeld() — Returns true every tick while UP is physically held down. Use for scroll acceleration or hold-to-repeat.

btnUpRel() — Returns true on the single tick when UP is released. Use for swipe-end or drag-end detection.
returns bool
Example — hold to scroll fast
static uint32_t heldSince = 0;

if (api->btnUp())     { scroll(-1); heldSince = api->now(); }
if (api->btnUpHeld() && api->elapsed(heldSince) > 600)
    scroll(-3);
bool btnSel()  ·  bool btnSelHeld()  ·  bool btnSelRel()
input
▶
The center/select button. btnSel() is the primary "confirm" action. btnSelHeld() is useful for long-press context actions. btnSelRel() fires when the button is released after a press.
returns bool
bool btnDown()  ·  bool btnDownHeld()  ·  bool btnDownRel()
input
▶
The down/next button. Mirrors the same three event types as btnUp but in the downward direction.
returns bool
OS shortcut Holding UP + DOWN simultaneously at any time exits the current app and returns to the launcher. Do not rely on that combination for in-app actions.

BLE & Messaging
9 functions
BLE is managed by the OS At boot the OS initialises BLE and calls AutoCoBle(). The main loop also blocks until a connection is established. Apps can check connection status and send commands, but do not need to manage scanning/connecting themselves under normal conditions.
bool bleIsConnected()
ble
▶
Returns true if a BLE connection to the Android companion app is currently active. Always check this before calling any messaging function.
returns bool
void bleOn()  |  void bleOff()
ble
▶
Initialises (bleOn) or fully shuts down (bleOff) the BLE stack. bleOff also disconnects any active session. Only call these if your app has a specific power management use-case.
void bleConnect(String name)  ·  void bleAutoConnect()  ·  void bleDisconnect()
ble
▶
bleConnect(name) — Scans and connects to a device by its advertised Bluetooth name.

bleAutoConnect() — Scans and connects to the first device advertising the NFX service UUID. This is what the OS uses at boot.

bleDisconnect() — Gracefully closes the active connection.
Example — reconnect flow
void onLoop() {
    if (!api->bleIsConnected()) {
        api->clear();
        api->text(30, 90, "Reconnecting...", 1);
        api->refresh();
        api->bleAutoConnect();
    }
}
void sendSMS(String phone, String msg)
messaging
▶
Instructs the Android companion app to send an SMS. Transmits phone + "|" + msg over BLE characteristic 0xFEED-0001. Does nothing if BLE is not connected.
ParamTypeDescription
phoneStringRecipient phone number as a string (e.g. "+33612345678").
msgStringMessage body. Keep under ~160 chars for a single SMS segment.
Example
if (api->btnSel() && api->bleIsConnected()) {
    api->sendSMS("+33600000000", "On my way!");
}
void loadConv(String phone)  |  void loadAllConvs()
messaging
▶
loadConv(phone) — Requests the conversation history for a specific number. The Android app will stream responses back via the notification characteristic.

loadAllConvs() — Requests the full list of SMS conversation threads. Responses arrive asynchronously over BLE notifications.
Response handling Responses from the Android app arrive via the BLE notify callback in BleManager.cpp. To use response data in your app you will need to expose it through a shared global or a callback hook — this is not yet part of the NFXAPI surface.

Guide — Building a Scrollable Menu

A pattern that covers most NFX apps: a scrollable list of items navigated with Up/Down and confirmed with Select.

App_Menu.h C++ 
#pragma once
#include "NFX_API.h"

class AppMenu : public App {
    NFXAPI* api;
    const char* items[5] = { "Inbox", "Contacts", "Compose", "Settings", "About" };
    int sel = 0;
    bool dirty = true;

    void draw() {
        api->clear();
        api->text(4, 4, "MENU", 1);
        api->line(0, 16, 200, 16);
        for (int i = 0; i < 5; i++) {
            int y = 24 + i * 20;
            if (i == sel) {
                api->rect(0, y - 2, 199, y + 12);
                api->text(8, y, items[i], 1, 0);
            } else {
                api->text(8, y, items[i], 1);
            }
        }
        api->refresh();
        dirty = false;
    }

public:
    String getName() override { return "Menu"; }

    void onStart(NFXAPI* a) override { api = a; sel = 0; dirty = true; draw(); }

    void onLoop() override {
        if (api->btnDown()) { sel = (sel + 1) % 5; dirty = true; }
        if (api->btnUp())   { sel = (sel + 4) % 5; dirty = true; }
        if (api->btnSel())  { /* launch sub-screen for items[sel] */ }
        if (dirty) draw();
    }

    void onStop() override {}
};

Guide — Quick SMS App

Send a pre-set SMS to a contact list using BLE, with connection status shown on screen.

App_QuickSMS.h C++ 
#pragma once
#include "NFX_API.h"

class AppQuickSMS : public App {
    NFXAPI* api;

    struct Contact { const char* name; const char* phone; };
    Contact contacts[3] = {
        { "Alice",   "+33600000001" },
        { "Bob",     "+33600000002" },
        { "Charlie", "+33600000003" }
    };
    int  sel    = 0;
    bool sent   = false;
    uint32_t sentAt = 0;

    void draw() {
        api->clear();
        if (!api->bleIsConnected()) {
            api->roundRectFill(10, 80, 190, 120, 6);
            api->text(30, 92, "No BLE connection", 1, 0);
            api->refresh();
            return;
        }
        api->text(4, 4, "Quick SMS", 1);
        api->line(0, 16, 200, 16);
        for (int i = 0; i < 3; i++) {
            int y = 28 + i * 22;
            if (i == sel) api->roundRectFill(2, y-2, 198, y+13, 4);
            api->text(10, y, contacts[i].name, 1, i == sel ? 0 : 1);
        }
        if (sent) api->text(40, 160, "Sent!", 2);
        api->refresh();
    }

public:
    String getName() override { return "Quick SMS"; }

    void onStart(NFXAPI* a) override { api = a; sel = 0; sent = false; draw(); }

    void onLoop() override {
        bool changed = false;
        if (api->btnDown()) { sel = (sel+1)%3; changed = true; }
        if (api->btnUp())   { sel = (sel+2)%3; changed = true; }
        if (api->btnSel() && api->bleIsConnected()) {
            api->sendSMS(contacts[sel].phone, "On my way!");
            sent = true; sentAt = api->now(); changed = true;
        }
        if (sent && api->elapsed(sentAt) > 3000) { sent = false; changed = true; }
        if (changed) draw();
    }

    void onStop() override { sent = false; }
};

Guide — Animation Loop

E-ink is slow to refresh, but you can still create effective animations by controlling update rate carefully.

App_Spinner.h — loading indicator C++ 
#pragma once
#include "NFX_API.h"

class AppSpinner : public App {
    NFXAPI* api;
    uint32_t lastFrame = 0;
    int angle = 0;
    const int R  = 30;
    const int CX = 100, CY = 100;
    const int FRAME_MS = 3000;  // new frame every 3 s

    void draw() {
        float rad = angle * 3.14159f / 180.0f;
        int   ex  = CX + (int)(R * cos(rad));
        int   ey  = CY + (int)(R * sin(rad));
        api->clear();
        api->circle(CX, CY, R);
        api->circleFill(ex, ey, 5);
        api->text(72, 140, "Loading...", 1);
        api->refresh();
    }

public:
    String getName() override { return "Spinner"; }

    void onStart(NFXAPI* a) override {
        api = a; angle = 0; lastFrame = a->now(); draw();
    }

    void onLoop() override {
        if (api->elapsed(lastFrame) >= FRAME_MS) {
            angle = (angle + 45) % 360;
            lastFrame = api->now();
            draw();
        }
    }

    void onStop() override {}
};
Tip Keep per-frame draw calls as minimal as possible. Only redraw regions that actually changed using targeted rect() + text() calls instead of clearing the whole screen every frame.
