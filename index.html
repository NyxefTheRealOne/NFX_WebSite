NFX OS abstracts the hardware into a clean API surface — NFXAPI* — so your app never touches GPIO, SPI, or memory directly. Write business logic, not drivers.

  • 01
    Sandboxed Runtime Apps operate exclusively through the NFXAPI interface. Direct hardware access is intentionally withheld.

    Partial Refresh v1.1 smart dirty-rect system computes the minimal changed rectangle — only redraws what changed.

JS Runtime

Upload any NFX OS .h app file. The emulator parses the C++ class, transpiles it to JavaScript, and runs it against a full browser implementation of the NFX API — including all drawing primitives, input handling, and the refresh cycle.

Use the three buttons to interact with your app just like the real device. UP+SEL+DOWN is not intercepted — use the Stop button instead.
